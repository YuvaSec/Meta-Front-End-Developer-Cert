
# Understanding Strings
- A string datatype is a collection of characters enclosed in ==' single '== or ==" double quotes "==.
- An empty string can be created using either single or double quotes, but it is not very versatile. ==' ' , " "==


# Creating and Using Strings
- Non-empty strings =  'hello there! 123'.
- ==Strings **cannot break onto the next line**; doing so results in an error. ==

# Common Issues with Strings
- Using an apostrophe within a string can cause errors; 
	- 'it=='==s is a lovely day' **//ERROR**
- to fix this, nest single quotes inside double quotes.
		- ==" it's a lovely day "==
		![[Screenshot 2025-11-29 at 15.21.27.png]]
- ***Consistency*** in using either single or double quotes is recommended to avoid confusion.
